03

사용자 시나리오User Scenarios

가입부터 리포트 출력까지의 동작 순서.

3-1회원가입 및 로그인

  1. 이메일로 회원가입(Firebase 이메일 인증), 도메인 제한 없음
  2. 가입 시 소속 부서 본인 선택
  3. 관리자 승인 전까지 앱 이용 불가(대기 상태)
  4. 승인 완료 시 알림 뱃지 수신, 로그인 가능

3-2소모품 신청 · 일반회원

  1. 품목 종류 선택 — 고정품목 / 추가품목 / 탕비위생품목
  2. 고정·탕비위생품목: 품목이름 또는 상품코드로 검색 후 선택
  3. 추가품목: 신규 품목정보 직접 입력, 고유번호 자동 생성(중복 방지)
  4. 품목·수량을 장바구니에 담아 한 번에 신청
  5. 단가 기준으로 신청금액 자동 계산
  6. 신청 완료 후 대기 상태로 등록
  7. 마이페이지에서 본인 신청현황·상태 확인
  8. 대기 상태 건은 본인이 직접 취소 가능
  9. 관리자 승인 시 알림 뱃지 수신

3-3신청 처리 · 관리자

  1. 부서별 통합 신청 현황 조회
  2. 신청 건(장바구니 단위) 처리
    • 승인 — 구매예정 내역서 대상으로 이동
    • 취소 — 반려 통보, 신청자가 수정 후 재제출 가능
    • 거절 — 신청 삭제
  3. 부서별 월 예산 한도 설정, 초과 시 경고 표시
  4. 구매 완료 처리
    • 입고수량 입력
    • 재고수량 자동 차감(수동 수정·엑셀 일괄 입력 지원)
    • 전체불출내역 데이터로 반영

3-4리포트 · 통계

  1. 부서별·품목별·기간별(주 / 월 / 분기 / 연) 통계를 표와 그래프로 확인
  2. 통합신청내역 부서별 엑셀 리포트 출력(월 지정 가능)
  3. 전체불출내역 PDF 리포트를 기간 선택 후 수동 생성·출력
04

핵심 기능 요구사항Core Requirements

9개 기능 영역.

4-1인증 · 회원관리

  • Firebase 이메일 회원가입 / 로그인
  • 가입 시 부서 선택 필수
  • 관리자 승인 전 서비스 이용 제한
  • 관리자 승격 기능(관리자가 관리자 지정)
  • 퇴사자 계정 비활성화 — 신청이력은 보존, 삭제하지 않음

4-2품목 관리

  • 고정 · 탕비위생품목 — 관리자 CRUD, 품목이름·상품코드·단위·단가 관리
  • 추가품목 — 신청 시 생성, 고유번호 자동생성 로직 필요, 해당 신청 건에만 종속(공용목록 미등록)

4-3신청 기능

  • 장바구니형 신청 — 여러 품목 동시 담기
  • 금액 자동계산 — 단가 × 수량
  • 상태 흐름 — 대기 → 승인 / 거절 / 취소
  • 본인 취소는 대기 상태에서만 가능
  • 신청현황 리스트 · 상세 조회

4-4승인 · 구매 프로세스

  • 신청 건 단위 일괄 승인·거절·취소 (품목별 개별승인 없음)
  • 승인 시 구매예정 내역서 자동 추가
  • 구매완료 처리 시 입고수량 입력 → 전체불출내역 반영

4-5재고 관리

  • 품목별 재고수량 관리 — 입고 · 사용 · 재고 자동계산
  • 승인·구매완료 처리 시 재고 자동 차감
  • 관리자 수동 수정 가능
  • 엑셀 일괄 업로드로 재고 일괄 반영
  • 최소재고 알림은 미적용(범위 제외)

4-6예산 관리

  • 부서별 월 예산 한도 설정
  • 초과 시 관리자 화면에 경고 표시

4-7알림

  • 인앱 알림 뱃지 방식 (웹푸시 아님)
  • 일반회원 — 승인 · 반려 · 거절 통보
  • 관리자 — 신규 신청 발생 통보

4-8리포트 · 출력

  • 통계 리포트 — 부서별 / 품목별 / 기간별 표 + 그래프
  • 통합신청내역 엑셀 출력 (월 지정, 부서별)
  • 전체불출내역 PDF 출력 (기간 선택, 수동 생성)

4-9데이터 정책

  • 연도가 바뀌면 새 연도 문서로 초기화 — 과거 데이터는 조회만 가능하며 삭제하지 않습니다.

주요 로직 명세Core Logic

6-1추가품목 고유번호 생성

  • 규칙 — EXT-{연도}-{일련번호 6자리} 형태 등 기존 items.itemId와 겹치지 않는 채번
  • 신청 저장 시 extraItems 컬렉션에서 최대 일련번호 조회 후 +1, 트랜잭션으로 중복 방지

6-2신청금액 자동계산

  • lineAmount = unitPrice × qty
  • requests.totalAmount = Σ requestItems.lineAmount

6-3예산 초과 체크

  • 승인 시점에 budgets.usedAmount += totalAmount
  • usedAmount > limitAmount이면 관리자 화면에 경고 배지 표시
승인 자체를 막지는 않습니다 — 경고만 노출합니다.

6-4재고 자동 차감

  • 구매완료 시 stockQty = stockQty + receivedQty − approvedQty
  • 관리자가 화면에서 수동 수정 가능 (변경이력 로그 권장)
  • 엑셀 일괄 업로드 시 상품코드 기준 매칭 후 일괄 갱신
